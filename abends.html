S001

Pode ter varias causas:

  • Data Check (incongruência em dados lidos/manipulados).
  • Fechar arquivo que não foi aberto ou Abrir arquivo que já esta aberto.
  • A DCB especificada no JCL não combina com a DCB real do arquivo.
  • Fita de 800 bpi's lida em unidade de 1600 bpi's ou vice-versa.
  • Fita 'SL' lida com 'NL'.
  • Especificação de unidade incompatível com o arquivo.
  • Fita para leitura com LABEL=(2,BPL), esta com a DCB diferente da especificada no programa.
  • Tentar ler (READ) um arquivo fechado (CLOSE).
  • O JCL foi catalogado sem os cartões SYSIN ou SYSOUT.
  • Falta codificar 'BLOCK CONTAINS 0 RECORDS'.
  • Erro na descrição da 'FD'.
  • Fita para leitura com LABEL=(2,BLP), esta com a DCB diferente da especificada no programa.
  • Tentar ler (READ) um arquivo após fecha-lo (CLOSE).
  • Arquivo de leitura esta mau gravado.
  • Os cartões de JCL foram catalogados sem os cartões SYSIN e ou SYSOUT.
  • Falta codificar 'BLOCK CONTAINS O RECORDS'
  • FD errada.
  • O DCB descrito não combina com o DCB do cartão de controle.

S002

Erro no processamento de um arquivo sequencial BSAM ou QSAM, ou na criação de um arquivo de organização direta. Uma ou mais das seguintes razões:

  • O Registro a ser transferido possuia mais de 32.767 bytes ou excedeu o tamanho da trilha, no caso de arquivo em disco.
  • Os Caracteres de controle de um registro de tamanho variável contem um valor fora do intervalo valido que vai de '4' a '32.767'. Isso também se aplica a blocos de tamanho variável.

Comprimento RECORD maior ou menor do que o OUTPUT.

S013

Erro na abertura de um arquivo, acompanhado da mensagem ICE141I:

  • RC = 1C - Erro de I/O no acesso ao diretório de um DATA SET PARTICIONADO.
  • RC = 08 - Cartucho ou Label errado.
  • RC = 10 - Tamanho de bloco não especificado no JCL ou o tamanho do registro descrito após a 'FD' não coincide com o tamanho real do arquivo.
  • RC = 13 - Não existe definição de DSCB e/ou não foi informada DCB na PROC.
  • RC = 18 - O membro de um DATA SET PARTICIONADO não foi encontrado. A mensagem especifica o STEP, o DATA SET PARTICIONADO e o MEMBRO.
  • RC = 20 - Erro de abertura de um arquivo QSAM causado por um ou mais dos motivos abaixo:
    • O BLKSIZE não é múltiplo de LRECL.
    • O BLKSIZE onde os registros são de tamanho variável não foi calculado com o acréscimo de 4 Bytes ao LRECL.
  • RC = 34 - Não existe definição de DSCB e/ou não foi informada DCB na PROC.
  • RC = 68 - BLKZISE maior que 32.767.

A DCB descrita no programa esta diferente da do arquivo de leitura ou gravação.

S033

Erro de I/O em um arquivo indexado sequencial.

Dica: Submeta o JOB novamente e se o erro persistir, refaça a criação do arquivo.

S038

Falta de espaço na área de índice de um arquivo indexado sequencial.

Dica: Apague o arquivo e refaça a sua criação, especificando uma área de índice maior.

S03A

Erro de I/O na VTOC durante o fechamento de um arquivo indexado sequencial.

Dica: Submeta o JOB novamente e se o erro persistir, recorra ao Suporte Técnico.

S03B

Para arquivo sequencial indexado.

  • O formato da DSCB indicando que o arquivo não será criado.
  • O arquivo não foi fechado ao ser criado.

S03E

Falta de espaço na área de dados de um arquivo indexado sequencial.

Dica: Apague o arquivo e refaça a sua criação, especificando uma área de dados maior.

S047

O Programa emitiu um SVC para um programa não catalogado em uma biblioteca valida, ou o programa chamado não foi catalogado com parâmetro de link-edição 'AC=1'.

S04E

Biblioteca autorizada movida de seu disco.

Utilitário preso no DB2.

S0B6

S0C1

Erro de operação (operation exception), causado usualmente por uma instrução de desvio errada.

Tome as seguintes providencias:

  • Verifique se no mapa de LINK-EDIÇÃO existem endereços que não foram resolvidos.
  • Verifique se existe um desvio para fora de um PERFORM.
  • Confira se foram especificados todos os cartões 'DD' dos arquivos em uso.
  • Verifique se o programa chama sub-programas e a opção 'ENDJOB' esta em efeito. As sub-rotinas do COBOL podem ter sido eliminadas da memória quando foi executado o 'EXIT' de um sub-programa chamado.
  • Verifique se os arquivos foram abertos de forma coerente com as operações efetuadas por ele. Um WRITE em um arquivo aberto como INPUT, por exemplo, poderia causar este ABEND.

S0C2

A Operação privilegiada e encontrada em estado problemático.

Cancelamento dentro do SORT antes do termino do arquivo.

S0C3

A instrução depende da execução de uma outra instrução.

S0C4

Erro de proteção (protection exception).

  • Pode ser causado pela referencia a dados de um arquivo, antes de ter sido processada uma leitura de registros deste arquivo.
  • Caso o item referenciado esteja definido na LINKAGE-SECTION, o erro pode ter sido causado pela passagem de parâmetros entre o programa principal e sub-programas.
  • Se o erro foi gerado em uma linha envolvendo um item de tabela, veja se o índice esta dentro dos limites da tabela.
  • Se o erro foi causado em uma tabela com um numero variável de ocorrências, verifique se o valor do campo descrito na clausula OCCURS NO TIMES DEPENDING ON variável esta correto.
  • Se o programa estiver processando registros de tamanho variável, verifique se o tamanho da área descrita sob a 'FD' e igual ao tamanho máximo do registro.
  • Uma SECTION pode não ter sido completada através de um EXIT.
  • Se o programa usa DISPLAY, veja se existe o cartão SYSOUT DD.
  • Se o programa usa ACCEPT, veja se existe o cartão SYSIN DD.
  1. DISPLAY no programa sem o cartão SYSOUT.
  2. ACCEPT no programa sem o cartão SYSIN.
  3. Estouro de tabela.
  4. Manuseio de campos com arquivos fechados. (CLOSE)
  5. Falta de EXIT em uma das SECTION'S.

S0C5

Erro de proteção (protection exception).

  • Pode ser causado pela referencia a dados de um arquivo, antes de ter sido processada uma leitura de registros deste arquivo.
  • Caso o item referenciado esteja definido na LINKAGE-SECTION, o erro pode ter sido causado pela passagem de parametros entre o programa principal e sub-programas.
  • Caso o comando que gerou o erro envolva item subscrito ou indexado, veja se o subscrito ou índice contem um valor incorreto.
  • Caso o dado referenciado seja de n de ocorrências variável, cheque se o valor do item do qual depende o n de ocorrências (occur depending on item) esta correto.
  • Veja também se o programa esta processando registros de tamanho variável como devido.
  • Pode ser também estouro de tabelas (montagem).
  1. Abrir (OPEN) ou fechar (CLOSE) 2 vezes um mesmo arquivo.
  2. Inverter os comandos entre os arquivos de entrada/saída. Ex: Mandar Gravar arquivo de entrada ou Mandar ler arquivo de saída.
  3. SORT - Falta de SECTION.
  4. Estouro de tabela.
